Chapitre 5: Système de Gestion de Version

Dans ce chapitre introductif, nous définissons Git et GitHub et expliquons l'importance des systèmes de gestion de versions pour la gestion de projets. Git est le leader incontesté des logiciels de gestion...

1) Introduction :

Git est le leader incontesté des logiciels de gestion de versions et est essentiel pour tout développeur.

2) A quoi sert concrètement un système de gestion de version ?

L'utilité d'un système de gestion de version à travers l'exemple d'un site web. Si vous êtes le seul à travailler sur le site, la sauvegarde et la restauration sont relativement simples, mais lorsque plusieurs développeurs travaillent sur le site, il devient complexe de sauvegarder et restaurer les modifications. Un système de gestion de version permet de résoudre ce problème en créant un serveur distant contenant l'historique des modifications effectuées par chaque développeur, et en permettant à chacun de travailler en local. Git est un logiciel de versioning qui permet aux développeurs de coordonner leur travail et de conserver un historique des changements effectués sur des fichiers, il aide les développeurs a savoir qui a effectué quel changement et quand, ainsi que les différences entre les différentes versions d'un fichier.

3) Les deux modèles des logiciels de gestion de version :

Les logiciels de gestion de version se basent sur deux modèles : le modèle centralisé> et le modèle décentralisé, également appelé modèle distribué. Dans le modèle centralisé, la source du code est hébergée sur un serveur distant central, tandis que dans le décentralisé, chaque utilisateur télécharge et héberge l'intégralité du code source sur sa propre machine. Git a popularisé le modèle décentralisé, qui présente plusieurs avantages, notamment la simplicité et la flexibilité du travail, ainsi qu'une meilleure sécurité grâce aux copies de sauvegarde de chaque utilisateur.

4) Qu'est-ce que GitHub ?

Git est un logiciel de gestion de version, qui permet d'enregistrer les différentes modifications effectuées sur un projet et de pouvoir retourner à une version précédente du projet, tandis que GitHub est un service en ligne qui permet d'héberger des dépôts Git et de servir de serveur central pour ces dépôts.

5) Utiliser Git : ligne de commande, console et interface graphique

Nous optons pour l'utilisation de la console et de la ligne de commande pour installer et utiliser Git, plutôt qu'une interface graphique. Ce choix est motivé par plusieurs raisons :

  • La ligne de commande permet d'avoir accès à toutes les commandes Git, contrairement à une interface graphique qui peut avoir des fonctionnalités limitées.
  • L'utilisation de la ligne de commande permet à tous les utilisateurs d'avoir accès aux mêmes commandes, ce qui garantit une uniformité dans l'apprentissage de Git.
  • Une fois que l'on sait utiliser la ligne de commande, il est facile de se familiariser avec n'importe quelle interface graphique.

En Windows, pour ouvrir la ligne de commande :

  1. Raccourci : Win + cmd
  2. Dans la barre de recherche, rechercher cmd

6) Installation de Git

Le processus d'installation de Git peut être simplifié en téléchargeant la dernière version sur le site officiel http://git-scm.com/downloads et en suivant les instructions à l'écran, en laissant les valeurs par défaut.

7) Paramétrage de Git

Configurer Git en renseignant un nom d'utilisateur et une adresse e-mail permet d'enregistrer ces données pour ne pas avoir à les fournir à nouveau plus tard. La commande Git pour effectuer cela est git config et l'option --global permet d'appliquer ces informations de manière globale pour tous les projets Git. Vous pouvez vérifier que les informations ont été enregistrées en tapant git config user.name et git config user.email dans la ligne de commande.

8) Démarrer un dépôt Git


Un dépôt Git est une copie de l'ensemble des fichiers d'un projet dans Git. Il y a deux façons de créer un dépôt Git : importer un répertoire existant dans Git ou cloner un dépôt Git existant.

9) La gestion des informations selon Git


Git stocke les données sous forme d'instantanés ou de "snapshots" de l'état du projet à un moment donné. Chaque instantané est stocké localement dans une base de données. Cette approche permet d'effectuer la plupart des opérations localement sans avoir besoin d'être connecté à un serveur distant, ce qui rend le travail plus rapide et plus agréable.

10) Les états des fichiers


Git suit les modifications de fichiers en les considérant comme soit "suivis" ou "non suivis". Les fichiers "suivis" peuvent être dans l'un des états suivants: "modifié", "indexé" ou "validé". Lorsqu'un dépôt Git est démarré, les fichiers sont non suivis et doivent être indexés et validés. Les fichiers modifiés doivent être indexés avant d'être validés dans un instantané, qui peut ensuite être recommencé.

11) Les zones de travail


Git utilise trois zones de travail : le répertoire de travail, la zone d'index et le répertoire Git. Le répertoire de travail est l'endroit où les fichiers du projet sont stockés et modifiés. La zone d'index contient les informations sur les fichiers qui feront partie du prochain instantané ou commit. Le répertoire Git stocke les méta-données et la base de données des objets du projet. Les fichiers peuvent être modifiés dans le répertoire de travail, puis indexés pour être inclus dans le prochain instantané et enfin validés pour être ajoutés au répertoire Git.

12) Créer un dépôt Git à partir d’un répertoire existant


Pour créer un dépôt Git à partir d'un répertoire existant, suivez ces étapes :
  1. Créer un répertoire pour le projet à versionner.
  2. Initialiser un dépôt Git dans ce répertoire à l'aide de la commande "git init".
  3. Utiliser la commande "git status" pour vérifier l'état des fichiers et répertoires du projet.
  4. Ajouter les fichiers et répertoires à l'index Git à l'aide de la commande "git add".
  5. Vérifier que les fichiers ont bien été ajoutés à l'index avec "git status".
  6. Valider les modifications en créant un nouveau commit avec la commande "git commit".
  7. Ajouter

13) Cloner un dépôt Git

Il est possible de créer un dépôt Git en clonant un dépôt existant déjà présent localement. Cette opération s'effectue à l'aide de la commande Git clone.

14) Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git

Le processus de gestion des versions avec Git nécessite que nous indiquions à Git tous les changements que nous apportons à notre projet, tels que l'ajout, la modification ou la suppression de fichiers. Pour ce faire, nous utilisons les commandes git add et git commit. Cependant, il est important de noter que le commit d'un fichier est basé sur l'état du fichier au moment du git add, ce qui signifie que les modifications apportées après le git add ne seront pas prises en compte. Pour enregistrer la dernière version d'un fichier, il est donc nécessaire d'effectuer un git add juste avant le git commit. Pour simplifier ce processus, nous pouvons utiliser la commande git commit -a pour ajouter et valider tous les fichiers modifiés et déjà sous suivi dans la zone d'index

15) Consulter l’historique des modifications Git

La commande "git log" permet de consulter l'historique des modifications réalisées avec Git. Cette commande affiche la liste des commits dans l'ordre chronologique, du plus récent au plus ancien, avec leur somme de contrôle SHA-1, le nom et l'e-mail de l'auteur, la date et le message du commit.

16) Annuler des modifications apportées à un fichier

Pour revenir à un état antérieur d'un projet enregistré avec Git, on peut utiliser la commande "roll back" pour annuler des modifications apportées à un fichier. Après un commit, si certaines modifications ne donnent pas les résultats escomptés, il est possible de revenir à l'état précédent du fichier en utilisant la commande git checkout -- nom-du-fichier ou la commande git restore.

17) Qu’est ce qu’une branche ?

Une branche dans Git est simplement un pointeur vers un commit spécifique.Git ne crée pas de copie physique de l'ensemble du répertoire de travail, ce qui rend la création de nouvelles branches et la fusion de branches très facile. La branche par défaut dans Git est appelée "master". qui est simplement un pointeur vers le dernier commit effectué sur cette branche. Lorsque nous créons une nouvelle branche, Git crée simplement un nouveau pointeur plutôt que de recopier l'intégralité du répertoire de travail
